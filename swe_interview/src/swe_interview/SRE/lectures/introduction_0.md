### Особенности курса

**Практическая направленность**:
- Курс основан на реальном опыте работы
- Примеры из текущей практики
- То, что применяется прямо сейчас

**Динамичность**:
- Курс сильно меняется каждый год
- Адаптируется под текущее состояние индустрии
- Возможны недоработки из-за постоянных обновлений

**Чего НЕ будет**:
- Подробного разбора конкретных технологий (Kubernetes, Prometheus, etc.)
- Идеально вылизанных слайдов
- Готового конспекта

**Что БУДЕТ**:
- Принципы, применимые к любым технологиям
- Зарисовки на экране для наглядности
- Реальные кейсы из практики

### Связь лекций и семинаров

| Лекции                | Семинары                             |    
|-----------------------|--------------------------------------|
| Принципы и концепции  | Конкретные технологии                |
| Как думать о системах | Как настроить Kubernetes, мониторинг |
| Почему так делать     | Практика применения                  |
| Универсальные подходы | Специфичные инструменты              |

**Пример**:
- Лекция: принципы построения отказоустойчивой архитектуры
- Семинар: настройка репликации в конкретной БД

### Применимость знаний

**Не только для IT**:
> Принципы работы с ненадёжностью, сбоями и непредсказуемостью применимы в любой сфере жизни

**Почему**:
- Жизнь всегда идёт не по плану
- Везде нужно работать с ошибками
- Принципы восстановления универсальны

**Деформация лектора**: "Я смотрю на жизнь через принципы SRE"

## Философия обучения

### Как правильно слушать курс

**Критическое vs Аналитическое мышление**:

❌ **Неправильно** (скептицизм):
```
"Я не согласен" → Закрываемся от новой информации
"Это неправильно" → Не слышим аргументы
"У нас делают по-другому" → Спор вместо обучения
```

✅ **Правильно** (аналитика):
```
"Интересно, почему лектор так считает?"
"Чем его подход отличается от моего?"
"Какие есть аргументы за и против?"
"В каких контекстах это работает?"
```

**Правило**: Если вы слышите только то, с чем согласны — вы ничего нового не узнаете.

### Что делать с расхождениями

**Ситуация**: То, что говорит лектор ≠ То, что принято у коллег

**Действия**:
1. ✅ Обсудить с лектором: "Почему так?"
2. ✅ Обсудить с коллегами: "Почему у нас по-другому?"
3. ✅ Проанализировать контекст применимости
4. ❌ НЕ спорить: "Нет, надо делать вот так!"

**Важно**: В нечётких областях (а SRE — нечёткая область) нет единственно правильного ответа.

### Ценность конспектирования

**Зачем писать конспект**:
- ❌ НЕ для того, чтобы перечитывать потом
- ✅ Сам процесс записи заставляет запомнить
- ✅ Структурирование информации в голове
- ✅ Формирование понятийного аппарата

**Эффект**:
> Чем больше вы знаете, тем легче гуглить/спрашивать ChatGPT. У вас есть база, на которую можно опереться.

## Оценка и экзамен

### Система оценивания

**Требования**:
1. Сначала закрыть семинары (балл > 3 из 10)
2. Только потом можно сдавать экзамен по лекциям

**Формат экзамена**:
- Письменные ответы на вопросы
- Вопросы из материала лекций
- Оценивается понимание принципов

**Автоматы**: Возможны для активных участников (но сложно при 110 студентах)

**Что НЕ даст высокий балл**:
```
"Скопировал из интернета"
"Переписал статью из блога"
"Попросил ChatGPT ответить"
```

**Что даст высокий балл**:
```
Собственное понимание принципов
Примеры из практики
Критический анализ подходов
```
